# Improve Python Trading System Workflow

When asked to improve the project, follow this loop:

1. inspect the relevant Python modules
2. summarize current behavior
3. identify top 3 highest-impact improvements
4. choose the safest and highest-value item
5. implement the change
6. add or update tests
7. update Markdown docs
8. summarize what changed
9. propose the next best improvement

## Decision priorities
Always prioritize:
- capital protection
- execution correctness
- broker reliability
- recoverability
- auditability
- maintainability

## Output expectations
For each run, provide:
- one completed improvement
- test coverage added
- documentation updated
- remaining risks
- next recommended task
